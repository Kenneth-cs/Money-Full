# 钱小满（Money Full）- 项目说明书与开发计划

## 项目概述
**钱小满（Money Full）** 是一款以“项目”为核心逻辑的创新型个人财务管理工具。告别传统的流水账模式，通过“超级收纳柜”（项目制）理念，将每一笔收支精确归属于特定生活事件、周期或目标中，带来极简、清晰的财务管理体验。

---

## 核心功能说明

1. **首页看板 (Dashboard)**
   - **功能用途**：展示全局视角的财务概况，提供最快的数据浏览入口。
   - **内容结构**：顶部展示本月总支出、收入与储蓄概况；中间展示正在进行中的项目卡片（含预算进度）；底部展示最近交易的流水列表。

2. **记一笔 (Add Record)**
   - **功能用途**：全局核心的快速记账入口。
   - **操作流**：点击悬浮大按钮 -> 切换收支 -> 选择分类 -> 选择归属项目 -> 输入金额（支持自定义丝滑数字键盘与基础计算） -> 完成。

3. **项目中心 (Projects)**
   - **功能用途**：集中管理所有的“抽屉”（即记账项目），如日常收支、海景房装修、新疆游等。
   - **操作流**：分为“进行中”和“已归档”两类。展示项目预算进度和统计信息，点击进入详情可查看该项目的专属时间轴（Timeline）。

4. **财务统计 (Analytics)**
   - **功能用途**：高级数据可视化中心，辅助用户进行财务健康度分析。
   - **展现形式**：包括环形图（按项目/分类占比）、折线图（支出趋势）、预算健康度色彩预警，以及智能财务洞察和建议。

5. **个人中心 (Profile)**
   - **功能用途**：账户管理与偏好设置。
   - **包含内容**：用户信息、数据概览（活跃项目数与总账目）、导出数据、主题设置等。

## 前端开发节奏规划 (Development Rhythm)

基于产品的 PRD 和 UI 稿件，为确保高质量交付，我们将 Flutter 前端开发分为以下 **五个核心里程碑** 进行推进，预计总周期为 **3-4 周**。

### 阶段一：项目初始化与底层架构搭建（第 1-3 天）
*目标：搭建坚固的基础设施，确保后续开发规范高效。*
- **脚手架与路由**：初始化 Flutter 项目，配置页面路由系统（如 GoRouter）。
- **状态管理**：引入适合中大型项目的状态管理方案（推荐 Riverpod 或 GetX）。
- **主题与UI规范**：基于 UI 设计稿提取莫兰迪色系渐变、极简白/深邃黑模式，封装全局统一的字体、颜色、卡片阴影、圆角等组件。
- **本地存储与数据模型**：设计并建立本地数据库（推荐 Isar 或 sqflite），完成 `Project` (项目)、`Transaction` (账单) 和 `Category` (分类) 的数据模型搭建。

### 阶段二：核心记账引擎 —— “记一笔”模块开发（第 4-7 天）
*目标：实现最核心的数据录入功能，追求极致丝滑的输入体验。*
- **页面框架**：实现底部弹窗/全屏模态页面的布局。
- **自定义键盘**：开发带有触觉反馈（Haptic Touch）、支持基础数学计算和日期补录的自定义丝滑数字键盘。
- **选择器联动**：实现支出/收入切换、分类网格（Category Grid）的滑动展示、归属项目（抽屉）的智能下拉选择。
- **数据联调**：实现记账数据的本地存储与更新。

### 阶段三：主视觉与信息流 —— “首页”与“项目中心”（第 8-12 天）
*目标：构建用户的核心视界，将冷冰冰的数据转化为有温度的卡片。*
- **首页看板**：开发莫兰迪绿色渐变财务看板，实现横向滑动的“进行中项目”视图，以及底部全局流水的列表渲染。
- **项目中心**：开发支持 Tab 切换的列表页面，实现内容丰富的项目卡片（预算双色进度条等）。
- **项目详情与 Timeline**：实现单个项目的详细流水时间轴呈现。

### 阶段四：数据可视化 —— “财务统计”与“个人中心”（第 13-17 天）
*目标：让数据说话，为用户提供直观的财务分析（深色模式为主）。*
- **图表引入与定制**：引入图表库（如 fl_chart），定制开发中心挖空的 Donut Chart（环形图）和贝塞尔曲线 Line Chart（折线图）。
- **预算预警与洞察**：实现色彩预警机制（进度超 80% 变橙，超 100% 变红），完成“智能洞察”文本卡片的 UI。
- **个人中心**：完成个人概览卡片和各类设置项入口（包含主题切换的逻辑联调）。

### 阶段五：体验打磨与进阶优化（第 18-20 天）
*目标：注入“高级感”，达到商业级 App 交付标准。*
- **动效与微交互**：为卡片展开、页面推入、进度条增长增加平滑过渡动画；补全各项操作的触觉反馈。
- **性能优化**：优化长列表滚动帧率、图表渲染性能。
- **测试与 Bug 修复**：覆盖核心场景的全链路测试。

## 技术选型说明
- **框架**: Flutter (iOS / Android 跨平台高度一致性)
- **语言**: Dart
- **UI 风格**: Clean & Minimalist, 卡片式设计, 莫兰迪色系渐变
- **数据存储**: 本地优先 (Offline First)，使用高性能本地数据库保障极致速度

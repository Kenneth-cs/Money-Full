# 产品需求文档 (PRD)：基于项目核心的记账 App

## 1. 产品概述
- **产品名称**: 钱小满（Money Full）*
- **产品定位**: 一款以“项目”为核心逻辑的创新型个人财务管理工具。
- **核心理念**: 告别传统记账 App 的“大储钱罐”模式（流水账），采用“超级收纳柜”（项目制）理念。将每一笔收支精确归属到特定的生活事件、周期或目标中。
- **技术栈**: Flutter（确保 iOS 和 Android 双端高度一致的体验与丝滑交互）。

## 2. 用户画像与核心场景

### 2.1 用户画像 (User Personas)
1. **生活规划达人 (The Organizer)**
   - **特征**: 喜欢将生活安排得井井有条，经常有阶段性的生活目标（如：海景房装修、婚礼筹备、考研计划）。
   - **痛点**: 传统记账把购买装修材料和日常买菜混在一起，难以直观看出特定计划的总体花费和预算执行情况。
2. **旅行爱好者 (The Traveler)**
   - **特征**: 每年有多次长短途旅行（如：新疆之旅、周末露营）。
   - **痛点**: 旅行期间消费频次高、类目杂，旅行结束后想快速盘点这趟旅行的总开销，且不希望突发的巨额旅行开销拉高“本月日常平均消费”的数据。
3. **斜杠青年/自由职业者 (The Freelancer)**
   - **特征**: 拥有多份收入来源，经常接不同的外包项目（如：品牌重塑项目、独立开发项目）。
   - **痛点**: 需要计算单个项目的“投入产出比（ROI）”，既有购买素材的成本，也有项目结款的收入，需要独立核算单项净利润。
4. **自律的新锐白领 (The Urban Professional)**
   - **特征**: 对个人现金流有清晰规划，注重“储蓄”与“日常开支”的平衡。
   - **痛点**: 想要一个干净、好看、无广告的记账工具，符合现代高级审美，且能满足日常记账极简、极速的要求。

### 2.2 核心使用场景 (Use Cases)
- **场景一：长周期大额项目的预算控制（如：海景房装修）**
  用户设立了“海景房装修”项目，总预算 75,000 元。在长达数月的装修期内，每一笔硬装、软装花销都记入此项目。App 统计模块会通过“色彩预警机制”实时提示预算进度（如达到进度 75% 时进度条变色），有效防止工程超支。
- **场景二：短周期高频支出的集中管理与归档（如：新疆之旅）**
  用户开启“新疆游”项目，在旅途中高频记录打车、门票、餐饮。系统默认为该项目准备了“特色小吃”、“租车”等专属标签。旅行结束后，用户一键将该项目“归档”，首页恢复清爽。但随时可在项目中心查阅这趟旅行专属的“时间轴（Timeline）”回忆账单。
- **场景三：带有收支双向流水的外包项目核算（如：品牌重塑项目）**
  用户接了一个设计私活。前期购买字体授权支出 120 元（记为该项目支出），后期收到客户尾款 5,000 元（记为该项目收入）。通过单项目统计，清晰得出该项目的净收益，且这笔流水的剧烈波动被完全锁在“抽屉”里，不影响用户对“日常吃穿用度”的数据分析。
- **场景四：纯粹的“日常开支”防污染（如：日常收支 Daily Living）**
  系统自带的常驻抽屉。用户平时的咖啡、通勤、三餐均常规记入此处。因为剥离了上述“大额、偶发”的项目开支，这里的统计数据能最真实、最平稳地反映用户的“基础生活成本”与“储蓄”健康度。

## 3. 核心业务逻辑
- **抽屉（项目，Projects）**: 记账的顶级容器。用户可以建立“日常开支”的常驻抽屉，也可以建立如“新疆之旅”、“海景房装修”、“同学生日派对”等短期抽屉。
- **账单（流水，Transactions）**: 每笔交易不仅需要打上分类标签（如：餐饮），还**必须**放进某一个指定的“抽屉”（项目）里。
- **双层分类标签系统**: 
  - **全局通用标签库**: 如“餐饮”、“交通”、“住宿”等，在所有项目里通用。
  - **项目专属标签库**: 例如在“新疆游”中新建“特产”、“门票”，这些标签仅属于该项目，不会污染日常记账的分类列表，保持全局清爽。
- **超级计算器（统计分析）**: 支持全局视角（所有抽屉的总账），也支持单一项目视角（单独拉出“新疆之旅”算细账）。

## 3. 功能模块详细说明 (严格参照 UI 原型)

App 采用经典的 4+1 底部导航布局（Bottom Navigation Bar）：首页、项目、中间悬浮的记账大按钮 (+)、统计、我的。

### 3.1 首页看板 (Dashboard)
首页提供全局视角的财务概况和高频操作入口。
- **顶部财务看板**: 
  - 采用清新的莫兰迪绿色渐变卡片。
  - 核心展示本月“当前总支出”（超大字号突出），以及“收入”和“储蓄”金额。
  - 右上角包含消息/通知铃铛图标。
- **进行中的项目 (横向滑动视图)**:
  - 核心区域展示当前活跃的项目卡片（如“新疆之旅”、“日常”等）。
  - **卡片内容**: 项目专属 Emoji/图标、状态标签（进行中）、项目名称、已用金额、预算金额、直观的横向进度条。
  - 右侧提供“查看全部”文字入口，点击可跳转至“项目中心”。
- **最近交易 (近期流水列表)**:
  - 按时间倒序排列的全局账单流。
  - **单条记录要素**: 圆形分类图标（带柔和背景色）、交易名称/备注、分类标签与时间（如：餐饮 · 今天 12:45）、金额（支出用显眼的红色如 `- ¥ 458.00`，收入用绿色如 `+ ¥ 5,000.00`）。

### 3.2 核心交互：记一笔 (Add Record - "+"悬浮大按钮)
放在屏幕最中间底部，强调记账的快捷与高效。
- **页面布局**: 底部弹窗或全屏模态页面，顶部带有“关闭(x)”和“完成”按钮。
- **收支切换**: 顶部的 支出/收入 胶囊式切换器。
- **金额输入区**: 超大号显眼色块（如醒目的黄色），展示超大字体的金额，带有光标动效。
- **分类选择 (Category Grid)**: 
  - 常用分类图标平铺排列（餐饮、购物、交通、居家等），支持点击“更多”或“设置”展开/编辑。
- **归属项目选择 (核心差异点)**:
  - 独立的下拉选择器（如默认选中“日常收支 (Daily Living)”）。
  - **智能默认**: 系统自动默认选择用户当前最常看或最近使用的那一个“项目”，用户可随时点击切换。
- **备注输入**: “添加备注...”的快捷文本框。
- **自定义丝滑数字键盘**:
  - 包含数字 0-9、小数点、退格键。
  - 包含日期选择图标（支持补账）。
  - 包含加减号（支持基础数学计算）。
  - 键盘区内嵌醒目的“完成”按钮，实现单手快速记账。

### 3.3 项目中心 (Projects)
对所有“抽屉”进行管理的专门页面。
- **Tab 切换**: 顶部提供“进行中”与“已归档”两个 Tab。旅游结束等短期项目归档后不再打扰首页，但数据永久保存。
- **项目列表卡片**:
  - 更大尺寸、信息更丰富的卡片式设计。
  - **内容要素**: 项目图标、项目名称、状态标签、创建日期（或周期描述）。
  - **项目描述**: 支持一段详细的文本描述（如“温馨自然的北欧风格...”）。
  - **预算模块**: 显示“预算进度”百分比、带双色填充的进度条、已用金额明细和剩余/总金额明细。
  - **交互**: 底部带有醒目的“查看详情 ->”按钮。进入详情页后，将展示专属于该项目的 **Timeline（时间轴）**，记录从项目第一天到最后一天发生的所有花费。
- **新建项目**: 列表最底部提供大面积的虚线框“+ 新建项目”按钮。

### 3.4 财务统计 (Analytics)
数据可视化中心（原型图展示为极具高级感的深色模式 / Dark Mode）。
- **时间筛选**: 顶部显眼的时间选择器（如“2023年10月”）。
- **项目占比 (Donut Chart 环形图)**:
  - 现代化平滑环形图，中心大字展示“总支出”。
  - 环形分色块展示，底部配合图例显示各项目或各分类的百分比（如：软件开发 42%、市场营销 42%）。
  - 支持全局视角（看各项目占总支出的比例）和项目内视角（看某个项目内吃、住、行的比例）。
- **项目趋势 (Line Chart 折线图)**:
  - 支持“日 / 月 / 年”维度切换。
  - 使用平滑的贝塞尔曲线折线图，配合渐变背景色，展示支出趋势。
- **多维度预算与健康度**:
  - 列表展示各个预算项（如办公租赁、市场营销、员工薪酬）的执行情况。
  - **色彩预警机制**: 当项目支出达到预算的一定阈值（如80%）时，进度条由绿色变为橙色；超支时变为红色，提供强烈的视觉预警反馈。
- **智能洞察 (Smart Insights)**:
  - **节省方案**: 给出基于数据的建议（如“市场营销支出连续三个月超出预算。建议整合...”）。
  - **健康提醒**: 给予正面激励（如“本月结余相比上月增长了12%...”）。
  - 提供“立即生成深度报告”的入口卡片。

### 3.5 个人中心 (Profile)
- **用户信息**: 头像（带编辑角标）、用户姓名/昵称、用户头衔/身份描述（如“高级财务分析师”）。
- **数据概览卡片**: 两个平行的统计卡片，展示“活跃项目”总数和“总账目”笔数。
- **账户管理菜单**:
  - 导出数据（支持将账单导出为便于查看的格式）。
  - 主题设置（切换深浅色模式等）。
  - 通知偏好。
  - 帮助与反馈。
- **退出登录**: 页面底部独立的红色强调按钮。

## 4. UI/UX 设计规范与高级体验要求

### 4.1 视觉风格 (Visual Style)
- **卡片式设计 (Card UI)**: 全局大范围采用圆角卡片，通过阴影和留白区分层级，界面干净整洁 (Clean & Minimalist)。
- **色彩体系**: 
  - 摒弃传统记账 App 的高饱和度大红大绿。
  - 背景以极简白（浅色模式）或深邃黑（深色模式）为主。
  - 点缀色和卡片背景采用**莫兰迪色系 / 轻盈色彩**的渐变色（如：新疆游使用沙漠黄渐变，日常使用薄荷绿渐变），使财务数据充满生活气息且不枯燥。

### 4.2 交互体验 (User Experience)
- **追求极速记账**: “记一笔”页面的键盘必须足够大，数字输入反馈必须丝滑无延迟。
- **触觉反馈 (Haptic Touch)**: 
  - 在点击底层导航栏、展开下拉菜单、特别是**点击统计图表（如滑动折线图节点）**时，必须加入细腻的震动反馈，提升应用的高级感。
- **动画过渡**: 卡片展开、页面推入、进度条增长等需加入平滑的过渡动画。

### 4.3 进阶功能规划 (Future-proofing)
- **语音记账接入**: 预留自然语言处理 (NLP) 接口，支持用户语音输入（例如说出：“今天新疆游机票花了1500”），系统自动识别金额填入，并自动匹配“新疆游”项目和“交通/机票”分类。
